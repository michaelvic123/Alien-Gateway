# zk-offchain-verifier

## Overview
This module provides a reusable Node.js verifier for Groth16 proofs generated by the Alien Gateway proof scripts. It validates proofs off-chain before submitting them to the Soroban contract, saving gas and preventing invalid submissions.

## Usage

### Import and Call
```ts
import { verifyProof } from './scripts/verify_proof';

const result = await verifyProof('non_inclusion', proof, publicSignals);
console.log(result.valid, result.message);
```

### CLI Test
To run the verifier test (after generating proof fixtures):

```
pnpm run test:verify
```

## Generating Proof Fixtures
To generate valid proof and public signals fixtures for testing:

1. Compile and set up the circuits:
   ```
   npm run compile:merkle_non_inclusion
   npm run setup:merkle_non_inclusion
   npm run compile:merkle_update
   npm run setup
   ```
2. Run the test scripts and modify them to save the generated proof and publicSignals:
   - In `zk/tests/test_non_inclusion_proof.js`, after proof generation, add:
     ```js
     require('fs').writeFileSync('tests/fixtures/non_inclusion_proof.json', JSON.stringify(proof, null, 2));
     require('fs').writeFileSync('tests/fixtures/non_inclusion_signals.json', JSON.stringify(publicSignals, null, 2));
     ```
   - In `zk/tests/test_update_proof.js`, after proof generation, add:
     ```js
     require('fs').writeFileSync('tests/fixtures/update_proof.json', JSON.stringify(proof, null, 2));
     require('fs').writeFileSync('tests/fixtures/update_signals.json', JSON.stringify(publicSignals, null, 2));
     ```
3. Run the scripts to generate the files:
   ```
   node tests/test_non_inclusion_proof.js
   node tests/test_update_proof.js
   ```

## Circuit Types Supported
- `non_inclusion`
- `update`
